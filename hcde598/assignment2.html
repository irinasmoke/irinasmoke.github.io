CAD Model + Fabricated Output

For this assignment, I recreated the shapes for the press fit construction kit from assignment 1, but with parameterized inputs so that I could easily and quickly adjust things like the size of the pieces and the notch width (useful for working with different materitals). To to this, I used Rhino for Mac.

Learning Rhino and Grasshopper basics

I started this assignment by getting familiar with the basics in Rhino and Grasshopper. I found it fairly straightforward to get the basic shapes from my original kit set up. However, some challenges I encountered were:

  • Looking up the names of commands without knowing what I should be Googling (e.g. "Fillet," "Scribble")
  • Understanding my own file. Keeping track of what items, particularly "Math" items, meant in Grasshopper

Cleaning up the file

The most challenging part of this project was keeping the grasshopper file tidy and keeping track of what elements corresponded to what. I started out working by not labeling items - e.g. having things named "Subtraction" or "Vector," but I would get very confused once I came back to the file after having not looked at it for a while. This led me to start labeling components with more meaningful names and grouping items, but I did not start doing this until the end. One learning for me is to keep a tidy file from the start!

Towards the end, I went though the file and cleaned it up by grouping things and renaming them. In this process, I found two points where I was unecessarily re-doing a calculation (I found this out when I went to label the items and realized I'd already labeled a very similar thing before).

Laser cutting iterations & Final Result

I used both thin and thick cardboard for this project. For the thick cardboard, I had already found a notch size that worked well from the last assignment, .15 inches, which was roughly the same as the width of the cardboard. Since it was corrugated, it had some "give," and therefore the notch size worked well.

For the thinner cardboard, I decided first to test out the same appraoch as with the thin - notch sizes equal to the width of the cardboard, or about .02 inches. This was way too thin. Because the thinner cardboard was not corrugated, it has very little "give." I tested .02, .03, and finally .04 inch notches. The last size I tried - .04 inches - worked well. Having the parameterized Rhino file made this process a lot easier than if I had to redo the shapes in Illustrator each time.

For the first couple iterations, the notches in the thin cardboard were too thin.

After adjusting the notch size, I printed kits in both the thin and thick cardboard. I decided to also play around with the size of the pieces, so I made the thin cardboard pieces smaller.

One thing I did not find useful was the final Array command I built into Grasshopper. I used this to repeat the elements, but I found that I wanted to set up my Illustrator files differently based on the size of cardboard that I had. I ended up Baking just one of each piece, exporting that to illustrator, adjusting the line thickness, and manually duplicating items there and placing them on the artboard as I saw fit.

I still had to do some manual adjustments in illustrator to optimize the use of space and fit the cardboard dimensions that I had.

Settings

I used the following settings for the thicker cardboard:
  • Power: 100%
  • Speed: 25%
  • Frequency: 50%
...and the following for the thinner cardboard:
  • Power: 50%
  • Speed: 50%
  • Frequency: 50%
